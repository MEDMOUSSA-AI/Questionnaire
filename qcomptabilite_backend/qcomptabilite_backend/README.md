# خادم استبيان: أثر المحاسبة الدولية على الأسواق المالية

خادم Flask بسيط يستقبل إجابات صفحة الاستبيان، يخزّنها في قاعدة بيانات SQLite،
ويوفّر لوحة إدارة (HTML/CSS) لعرض الإجابات وتنزيلها كملف Word بضغطة واحدة.

## الملفات

```
qcomptabilite_backend/
├── app.py              # التطبيق الرئيسي (يقدّم صفحة الاستبيان + المسارات + قاعدة البيانات + تصدير Word)
├── requirements.txt
├── Procfile             # لتشغيل التطبيق على Render عبر gunicorn
├── templates/
│   ├── index.html         # صفحة الاستبيان (عربي/فرنسي) — تُعرض على الرابط الرئيسي "/"
│   ├── login.html         # صفحة تسجيل دخول الإدارة
│   ├── dashboard.html     # قائمة كل الإجابات
│   └── detail.html         # تفاصيل إجابة واحدة
└── responses.db          # تُنشأ تلقائيًا عند أول تشغيل (قاعدة بيانات SQLite)
```

كل شيء (صفحة الاستبيان + الخادم + لوحة الإدارة) في تطبيق واحد، تنشره كخدمة واحدة على Render.

## التشغيل محليًا

```bash
cd qcomptabilite_backend
pip install -r requirements.txt
python app.py
```

سيعمل الخادم على `http://localhost:5000`:
- صفحة الاستبيان: `http://localhost:5000/`
- لوحة الإدارة: `http://localhost:5000/admin`
  (اسم المستخدم الافتراضي `admin`، كلمة المرور الافتراضية `changeme123` — **غيّرها قبل النشر**)

## النشر على Render

1. أنشئ مستودع GitHub جديد وارفع محتوى هذا المجلد إليه (بنفس الطريقة التي اتبعتها مع `bourse-nouakchott`)،
   أو ارفع ملف الـ zip مباشرة إذا كانت طريقة النشر التي تستخدمها تدعم ذلك.
2. من لوحة Render: **New → Web Service** → اربط المستودع.
3. الإعدادات:
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `gunicorn app:app` (موجود مسبقًا في `Procfile`)
4. أضف متغيرات البيئة التالية من تبويب **Environment**:
   | المتغير | القيمة المقترحة |
   |---|---|
   | `SECRET_KEY` | نص عشوائي طويل وسري |
   | `ADMIN_USERNAME` | اسم المستخدم الذي تريده |
   | `ADMIN_PASSWORD` | كلمة مرور قوية |
5. بعد النشر، رابط مثل `https://qcomptabilite-backend.onrender.com` سيعرض صفحة الاستبيان مباشرة،
   ولوحة الإدارة على `/admin` — لا حاجة لأي إعداد إضافي أو ربط يدوي بين ملفين منفصلين.

### ⚠️ ملاحظة مهمة حول قاعدة البيانات على Render (الخطة المجانية)

قاعدة البيانات SQLite (`responses.db`) تُخزَّن كملف على القرص. في الخطة المجانية لـ Render،
هذا القرص **غير دائم** (ephemeral) — أي أن الملف قد يُحذف عند إعادة تشغيل الخدمة (مثلاً بعد فترة
خمول طويلة). للحفاظ على البيانات بشكل دائم لديك خياران:

- **الأسهل:** أضف *Persistent Disk* من إعدادات الخدمة في Render (متاح حتى في بعض الخطط المدفوعة الرخيصة)
  واربط المسار بـ `DB_PATH` كمتغير بيئة يشير إلى مجلد القرص الدائم.
- **الأفضل لبحث أكاديمي طويل المدى:** استبدل SQLite بقاعدة PostgreSQL (كما فعلت في مشاريعك الأخرى)
  — أخبرني إن رغبت وسأجهّز لك النسخة المكافئة بـ PostgreSQL.

## نقاط النهاية (Endpoints)
| `GET /api/health` | فحص أن الخادم يعمل |
| `GET /admin` | لوحة الإدارة (تتطلب تسجيل دخول) |
| `GET /admin/response/<id>` | تفاصيل إجابة واحدة |
| `GET /admin/response/<id>/word` | تنزيل تلك الإجابة كملف Word |
| `GET /admin/export-all/word` | تنزيل كل الإجابات في ملف Word واحد |
| `POST /admin/response/<id>/delete` | حذف إجابة |
